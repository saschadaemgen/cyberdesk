CYBERDESK ROADMAP (soft cuts, status 2026-07-08)

Principles:
- Risk first: the unproven moves to the front.
- Security before integration: no door control before authorization stands.
- D-0006: The better, harder path beats the compromise. Governs the quality of the path, not all-at-once scope.
- Season end: season protocol (including failures and dead ends) + chat handover + update of the five living docs. Stored in seasons/ AND in the chat project.

S1  Foundation + risk proof     CD-01 shell+CEF [running], CD-02 OSR texture, CD-03 feathering
S2  Design law                  Color world, motion language, shaders, drag-and-drop language (CD reference binding)
S3  Zone engine                 Grid, S/M/L reflow, tab rail, Edit Mode
S4  Modes + Event Engine        Standard/Admin/DND, priority x mode, simulated events
S5  Browser becomes CARVILON    Favorites/history (SQLite), downloads, context menu, request filter
S6  Crypto + authorization      Argon2id, Zeroize, encrypted state, start authorization
S7  CARVILON integration        Cameras, doors, status, time clock (NetGuard rules Edge/VPS)
S8+ Tools                       Terminal, editor, explorer, FTP, logic analyzer, NetGuard monitor
S9+ Everyday                    Music, SimpleX, email, calls, calendar/day planner, office unit (text+tables in one)

Long-term goal: CARVILON OS (Debian 13 Trixie) - CyberDesk boots as the shell. Nothing on the app path is throwaway work.

Note: Numbers are orientation, not a contract. Chat fill level and reality decide the actual cuts.
