# CyberDesk - Architecture

## What CyberDesk is

A single fullscreen application in the style of a serious cyber operating system: fixed zone layout, one color world (base #04070A, brand blue #009FE3), heavily animated, optimized for ultrawide displays (target roughly 1.20 m of screen width, 16:9 as fallback). Only CARVILON-built applications run inside it, plus one surf zone. No freely movable windows - predictability is the core of the product.

## Layer model

1. **Zones (fixed):** Spine on the left, main area (three sizes S/M/L with reflow of neighboring zones), video zone, terminal zone, right tab rail (Status, Files, FTP, Music, ...). Positions are law. Changes only in Edit Mode within the grid rules (D-0007).
2. **Modes (layout presets):** Standard, Admin (larger main area), Do-Not-Disturb, and others. A mode loads a preset of the same fixed layout - it never invents a new one.
3. **Event Priority Engine:** Events (doorbell, call, alarm, security alert) carry priorities, zones carry ranks, the mode acts as the gate. Decision per event: override, overlay, or suppress. Even interruptions are regulated.

## Process and technology model

- **Rust host:** window management (winit), rendering (wgpu), zones/modes/event engine, later crypto (Argon2id, Zeroize) and start authorization.
- **CEF (Chromium Embedded Framework):** delivers pixels of the surf zone, nothing else. CD-01: windowed embed as feasibility proof; from CD-02 on, offscreen rendering into a GPU texture, then feathering/compositing inside our own frame (soft edges bleeding into the design). CEF runs with an isolated browser profile (own root_cache_path) - the surf zone never shares state with any user-installed browser.
- **Hard process boundary host<->CEF**, IPC only through an explicit allowlist. No Electron, no Node, no npm chain in the core. Chromium sandbox stays active.
- **NetGuard:** no module opens connections on its own; everything goes through the central network layer (deny-by-default per zone, certificate pinning, own DNS resolver, kill switch, counters). Browser traffic attaches to the same monitor via CefRequestHandler. **Two documented exceptions exist:** (D-0023, CD-13) the update-awareness checker fetches a single pinned CARVILON manifest URL over HTTPS; and (D-0027, CD-15) per-window Tor traffic — user-driven browsing through the Tor network when a window is toggled to Tor (embedded arti, a local per-slot SOCKS relay). Both are the host's only intentional outbound paths until NetGuard is built, when they become its first allowlist entries.

## Platform path

Development: Windows 11 (MSVC). Later: Linux appliance. Long-term goal: CARVILON OS (Debian 13 "Trixie") booting directly into CyberDesk as its shell - the app is the first deliverable and the later heart of the OS. Nothing on the app path is throwaway work.

## Status

Season 1 extended complete; Season 2 begun (CD-12). CD-01: shell (winit/wgpu, borderless fullscreen, rotating ring) plus a chromeless CEF windowed embed of google.com. CD-02: off-screen rendering - the page becomes a wgpu texture composited inside our own frame (CPU path; accelerated path researched, D-0009). CD-03: feathered surf-zone edges, the procedural Deep Field background, and a web-isolated `cyberdesk://` settings view over a message-router IPC bridge (D-0010). CD-04: free surfing - a `Ctrl+L` command bar (host-side URL-vs-search), history navigation (Alt+arrows, mouse buttons 4/5, F5 / Ctrl+R / Ctrl+Shift+R), a gesture-aware popup policy (D-0011), a loading line, and a tier-1 foreground guard. CD-05: background v2 - the seeded, bake-once "Pulse Grid" circuit board (traces, pads, bus lines, travelling pulses, node flares) replaces the Deep Field as the Cyber default and is allowed to glow, with a zone shadow dimming it under content and a live glow-intensity slider; the Deep Field is demoted to a token-selectable "Calm" variant (D-0012). CD-06: design corrections (D-0013) - the ring is removed from the shell (its motif migrates to the Season-2 start animation / Energy Core), the Pulse Grid becomes three depth layers (far/mid/near, ~10x the content at ~12k baked primitives, with chip footprints, via clusters and junction hubs) baked into the one HDR texture, pulses live across all depths, and the surf-zone feather is narrowed to a light, steep edge soften; push policy becomes autonomous per stage. CD-07: the daily driver - local memory (a schema-v2 SQLite `history` + `favorites` store, capped at 10k rows) turns the command bar into a command palette with live frecency-ranked suggestions from favorites and history, `Ctrl+D` favorites the current page, and the Google-only search fallback becomes a chosen search engine (Google/DuckDuckGo/Bing/Startpage) via the settings page's first select control (D-0014, D-0015). CD-08: the command surface moves from a centered palette to a **hover-reveal top bar** in the gap above the surf zone - it slides down on mouse-to-top or `Ctrl+L` (host-side scissor-clipped slide, ~180 ms ease-out) and retreats on mouse-out (~250 ms hysteresis, with a typing exception), a committed navigation, or `ESC` (chain: bar -> settings -> quit), showing favorites as clickable **chips** beside the address input; this revises D-0014's "no favorites bar" (D-0016). A measured display fix also ends the CD-07 "only one favorite shows" bug (the palette had filtered favorites by the prefilled current URL). CD-09: the multi-slot engine (D-0017) - the single surf zone becomes up to four **fixed-width content columns** ("slots", 1200 logical px, gutter-spaced, group-centered; `max_slots` fits the width - four on 5120, one on 1920). Slots are lazy (a placeholder with an index glyph until the first navigation, no white flash) and keep a stable id (browsers/textures never migrate between indices; only display position changes). One slot is active (a bottom accent line, sole CEF focus); `Ctrl+T`/`Ctrl+W` add/close, `Ctrl+1..4`/`Ctrl+Tab` switch, keyboard drives the active slot and the mouse drives the slot under the cursor (a click activates it). The page pass is now shared-pipeline + per-slot targets with an instanced placeholder pass and an instanced slot-lines pass (loading line + active accent); the zone-shadow uniform grew to 6 rects. The top bar targets the active slot with no new IPC. The D-0009 gate was measured at 4 slots on the 5120 ultrawide (full frame median ~4.5 ms, worst ~6.8 ms - well inside the 60 fps budget): the stutter trigger did not fire, the CPU OSR path stays, and the accelerated zero-copy path stays the recommended-but-not-yet-needed next optimization (D-0017). CD-10: slot polish (D-0018, D-0019) - the workspace becomes permanent and fluid. A schema-v3 `session_slots` table persists the ordered slots (url/width/active), written wholesale + debounced on change and restored on launch (fit-from-left by width units, overflow kept for a wider restart); the active column reloads immediately, the rest stay lazy with their URL pre-armed (a scheme-colored placeholder dot marks a pending page) and spawn on first touch. Slots gain **width units** (`Ctrl+Shift+D` toggles single/double, `slot_rects_units` keeps a U-unit group the same extent as U columns, all rendering/routing unit-agnostic) and **rearrange** (`Ctrl+Shift+Left/Right`, a pure order swap, no browser move - hard swap, the animation deferred to the Edit-Mode drag language). The D-0011 popup policy is extended (D-0018): a user-gesture popup (target=_blank, or a Ctrl/middle-click - Chromium routes modified clicks through `on_before_popup` as tab dispositions) opens a new column beside the source when there is room, else navigates in place; ad popups stay dead. CD-11: the main frame (D-0020) - the slot group no longer owns the full width. **Side zones** flank it left and right (placeholder now: subtle fill, thin outline, a diamond glyph; the Spine / status-files-music rails live here later), first-class geometry via a pure `slots::frame_layout` that reuses CD-10's `slot_rects_units` (the symmetric frame keeps the group centered). The reflow law: **Full** side zones (320) when the slots fit with them, else retreat to thin **rails** (48); capacity (`frame_capacity`) is the rail center budget, so side zones reduce mid-size capacity (2560→1 column) while the 5120 ultrawide still reaches four (1-3 full sides, 4 rails). The `gutter` token widened 24→40 into reserved control territory (CD-12 drop zones). The reflow is **animated and desync-safe by construction** (pre-empting the D-0019 hard-swap precedent): one per-frame animated frame - per-slot rects ease to the target, a single `side_width` drives the sides - and both rendering AND input read the same geometry, so it is one fluid ~220ms motion, never a jump. The zone-shadow uniform grew 6→8 rects. CD-12: floating command elements (D-0021) - the single top bar is **retired**. Every column carries its own floating **command set** (back/forward/reload orbs + an address capsule) that reveals above it and drives it; favorites become **round tiles** in one shared launcher. All of it floats on ONE transparent OSR band view (background_color 0x00000000, premultiplied) so only the pills paint and the Pulse Grid breathes between them - the band casts no zone shadow. The host supplies **frame state** (engaged slot + each column's band-DIP x/width) pushed **on change only** via `window.cdFrame` (the CD-11 cadence; the page glides via CSS ~220ms, so a trailing ensemble during reflow is correct), and the page pulls `get_frame` on load; the nav/palette commands gained an optional `slot` field so each ensemble targets its own column (`target_slot`). The CD-08 reveal state machine generalizes to N (hover-a-gap engages that column, Ctrl+L the active one, hysteresis/typing/linger unchanged). **Dragging a favorite tile into a control gutter** opens it there as a new column: the page fires only `drag_start`, the HOST owns the drag (shell-drawn ghost + gutter drop zones, mouse captured, ESC cancels; at full capacity it navigates the slot under the ghost). Each column also reveals a shell-drawn **close orb** (a ring + cross at its top-outer corner) that closes it. The drag visuals and close orbs share one instanced overlay pass (`drag.wgsl` + `DragOverlay`, a `kind` flag switches filled-rect vs ring+cross, drawn topmost). Revised frame law (D-0022, supersedes the symmetric-zones + four-slot parts of D-0020): four columns were too many and the zones matter more, so the **slot maximum drops to THREE** (a `slot_max` token caps capacity / count / restore everywhere; `MAX_SLOTS` stays 4 as the array ceiling) and the frame becomes **asymmetric** — the RIGHT is the **permanent Multifunctional (MF) zone** (never rails, a three-bar rows glyph; the future tab area) and the LEFT (Spine, a diamond) is the flexible one that retreats to a rail. The whole frame is centered as a block, so with the left railed the group shifts by `(left_width - mf_width)/2` toward it (reusing `slot_rects_units` + a translation; the shell eases the LEFT width, the MF width is constant); render + input still read one per-frame geometry (desync-safe). The `gutter` rose 40→56; capacities became 1920→1, 2560→1, ~3000→2, 5120→3 (two slots now need ~3000). The CD-12 floating layer adapted with no change (it reads frame rects generically). CD-13: the info area (D-0023, D-0024) - update awareness, and the host's FIRST intentional outbound connection. A background worker fetches ONE pinned CARVILON manifest over HTTPS (NetGuard exception D-0004→D-0023: one allowlisted URL from a config token, ureq/rustls, hard timeouts, silent on failure), compares the running versions (CyberDesk `CARGO_PKG_VERSION`; CEF/Chromium from the pinned crate's compile-time constants) and produces generic info items. Rendered per the floating law (D-0021): a shell-drawn info glyph beside the gear (idle ring / filled+pulse+count when updates exist) and a floating `cyberdesk://info/` panel (a new mutually-exclusive overlay, ESC-chained), listing the CyberDesk + CEF items with release-notes links (open in a slot) and versioned Dismiss. Store schema v4 caches the manifest + last-check + dismissals. The item model is deliberately the seed of the Season-7 notification rail; V1 **informs only** - self-update arrives with the signed ML-DSA pipeline (Season 6+), and this surface is where its Install action will live (D-0024). CD-14: the own start page + privacy defaults (D-0025) - Google is banished. Every empty slot opens to an internal `cyberdesk://start/` page served from the binary (zero network, same isolation as settings/info): a black canvas with the reserved Energy Core motif (CD-06 - concentric rotating brand arcs around a bright hollow core, prefers-reduced-motion aware), a search capsule (reusing the host URL-vs-search classifier + search_engine), and favorite tiles. The slot's start page uses `cefQuery` - the browser-side router now forwards for all views (safe: `cefQuery` is exposed only on `cyberdesk://` frames). And **websites are not saved**: the CD-10 session-URL persistence is reversed for privacy (schema v5 drops `session_slots`, purging any prior URLs); `restore_session` unconditionally boots one start-page slot, so a restart never restores open pages (history + favorites untouched). Big-monitor focus this season (sub-1920 polish deferred). CD-15: per-window Tor (D-0026 engine, D-0027 per-window proxy) - clearnet is the default; each column has its own Tor toggle. The engine is embedded **arti** (pure-Rust Tor) on a background tokio runtime exposing a per-slot local SOCKS5 relay (127.0.0.1, one isolated arti client per slot → unlinkable circuits, remote DNS); bootstrap is off the shell thread (status atomic, never freezes). Switching is per-`CefRequestContext`: a Tor slot's browser is created under a context whose `proxy` pref is its SOCKS port plus the WebRTC leak prefs (QUIC off globally), created UI-thread via post_task; a clearnet slot uses the direct global context; the toggle respawns the browser under the other context at the start page. The wiring is **fail-closed** (no direct-connection fallback; popups inherit the source's Tor mode; a mode-mismatched browser is closed in on_after_created) — three real fail-open IP leaks were caught and closed by an adversarial security review before shipping. Settings hold the engine switch + a "Tor for new windows" default + a live status readout, with a mandatory **honesty label** (hides the IP, but NOT Tor-Browser-grade anti-fingerprinting / no TLS-layer change). SBOM/CRA: `arti-client`, `tokio`, `tokio-util`, `tor-rtcompat` (+ their tree) are new shipped dependencies (D-0026). The routing/leak checklist (check.torproject.org, DNS-leak, WebRTC, two-circuits) is verified live on the maintainer's networked machine — stated, not assumed.
